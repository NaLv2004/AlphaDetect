
# [Research Prompt: 自动化科学发现] 
# 主题：Emergent Algorithmic Structures in MIMO Detection via Typed Stack-Based Genetic Programming

## 1. 核心目标与研究哲学 (Core Objective & Philosophy)
你需要从零开始编写一个极其硬核的**自动化算法发现系统（Minimum Working Example, MWE）**。
目标是解决通信系统中的 MIMO 检测问题 ($\mathbf{y} = \mathbf{Hx} + \mathbf{n}$)。
**核心限制（绝对不可违反）：** 
绝对禁止在代码中硬编码或通过接口预定义任何高级算法结构（如 BP、MMSE、深度为 K 的树搜索等）。机器必须通过组合极底层的**代数算子**、**图论算子**和**抽象控制流**，在一个完全空白的“评估孔洞 (Hole)”中，自发涌现出高级计算图（例如在动态生成的搜索树上运行局部的消息传递以改进启发式度量）。

## 2. 外部宏观框架：带有 QR 分解的 Stack Decoder
系统外层由一个死板的、确定性的 Best-First Search (Stack Decoder) 驱动。
*   **物理预处理：** 给定 $\mathbf{H}$ 和 $\mathbf{y}$，执行 QR 分解 $\mathbf{H} = \mathbf{QR}$。计算 $\tilde{\mathbf{y}} = \mathbf{Q}^H\mathbf{y}$。搜索基于上三角矩阵 $\mathbf{R}$ 从底层（第 $N_t$ 层）向顶层（第 1 层）展开。
*   **搜索树维护：** Stack Decoder 维护一个优先队列（Priority Queue）和一棵**动态生长的搜索树 (Search Tree)**。每个节点包含：当前层数 $k$、已决定的符号序列 $\mathbf{x}_{[k:N_t]}$、以及当前节点的局部累计欧氏距离。
*   **算法孔洞 (The Hole)：** 每次从优先队列中弹出一个节点准备扩展时，如何评估其子节点的优先级（Score）是**未知**的。Stack Decoder 会调用 `emergent_metric(Environment_Snapshot)`。返回的 Float 越小，优先级越高。

## 3. 内部微观框架：MIMO-Push 虚拟机 (The PushGP VM)
`emergent_metric` 是由 PushGP 虚拟机执行的一段随机演化的堆栈代码。VM 不使用任何变量，只有类型化的堆栈（Stacks）。

### 3.1 堆栈定义 (The Stacks)
系统必须维护以下数据栈（Data Stacks）和控制栈（Control Stacks）：
*   `Float`, `Int`, `Bool` (基础标量栈)
*   `Vector`, `Matrix` (张量栈)
*   `Graph` (图结构栈)
*   `Node` (图节点指针栈)
*   `Exec`, `Code` (用于实现图灵完备控制流的指令栈)

### 3.2 环境快照注入 (Environment Snapshot Injection)
每次调用 `emergent_metric` 前，清空所有栈，并将当前物理状态压入：
*   `Matrix` 栈顶：压入 $\mathbf{R}$。
*   `Vector` 栈顶：压入 $\tilde{\mathbf{y}}$，压入当前已假设的部分符号向量 $\mathbf{x}_{\text{partial}}$。
*   `Graph` 栈顶：压入**整棵当前的动态搜索树 $G$**。
*   `Node` 栈顶：压入**当前正在被评估的候选子节点 $n_{\text{cand}}$**。
*   `Int` 栈顶：压入当前节点所处的深度 $k$。

## 4. 极端抽象的底层算子库 (Primitive Operators)
算子必须极其原子化。严格禁止包含特定物理意义。

*   **栈操作指令：** `Pop`, `Dup` (复制栈顶), `Swap`, `Flush` (清空某栈)。
*   **张量代数指令：**
    *   `Mat.Mul`, `Mat.VecMul`, `Vec.Add`, `Vec.Sub`, `Vec.Dot`, `Vec.Norm2`
    *   `Mat.InvertSafe` (计算伪逆，若奇异则返回单位阵)
    *   `Mat.ExtractTopLeft`: 弹出 Int $k$ 和 Matrix $\mathbf{M}$，提取 $k \times k$ 左上角子阵。
    *   `Vec.ExtractSlice`: 弹出 Int $k$ 和 Vector $\mathbf{v}$，提取前 $k$ 个元素。
*   **图拓扑与漫游指令：**
    *   `Graph.GetRoot`, `Node.GetParent`, `Node.GetChildren` (将邻居节点群压入一个临时的列表或序列中执行映射)。
    *   `Node.GetData`: 弹出 Node，将该节点上挂载的张量信息（如当前符号、累计误差）压入 `Vector` 或 `Float` 栈。
*   **控制流与作用域指令（完全抽象的涌现源泉）：**
    *   `Exec.If`: 弹出 Bool 栈顶，若为 True 执行 `Code` 栈顶代码块，否则执行次栈顶代码块。
    *   `Exec.While`: 弹出 `Code` 栈顶作为循环体。只要 `Bool` 栈顶为 True，就持续在 `Exec` 栈展开循环体。
    *   `Graph.MapReduce`: 这是一条高阶控制流。弹出 `Code` 栈的一段代码，将其独立作用于当前 `Node` 栈顶节点的所有子节点/邻居节点。将其产生的 `Float` 栈顶结果相加/相乘。**(这是机器自发产生 Message Passing / BP 的核心机制，机器需自己决定 Map 的代码块是什么，以及向多深的邻居去 Map，而不是被硬编码深度)**。

## 5. 演化引擎与严格防爆机制 (Fitness & Constraints)
因为控制流完全由机器利用 `Exec.While` 和图遍历动态组合，系统极易陷入死循环或遍历整个 $2^{N_t}$ 的树，导致内存或时间爆炸。必须建立极度严格的约束机制。

### 5.1 硬件级 FLOPs 计数器 (The FLOPs Counter)
*   VM 内部强制集成 `flops_count`。
*   执行 `Vec.Add` 消耗 $O(N)$，`Mat.Mul` 消耗 $O(N^3)$。
*   如果执行过程中 `flops_count` 超过阈值 $T_{\text{max}}$（例如 $10 \times N_t^3$），VM 立即引发 `Hardware_Fault` 中断。清空栈，强行将 `Float` 栈底置为 $\infty$（最差得分）。这种机制将**倒逼机器通过 `Exec.If` 和动态条件生成局部的树遍历（只遍历部分节点）**，而非全局遍历。

### 5.2 多目标适应度评估 (Lexicase / Pareto Fitness)
给定一个小规模 MIMO 数据集（如 $4 \times 4$ QPSK，无噪声和中等噪声混合）：
1.  **Metric 1 (Performance):** MSE = $\|\hat{\mathbf{x}}_{\text{stack}} - \mathbf{x}_{\text{ML}}\|^2$ （与最大似然解的均方误差，或者 BER）。
2.  **Metric 2 (Complexity):** 成功解出该局部的平均耗费 FLOPs。

在演化选择（Selection）阶段，优先淘汰触碰 FLOPs 上限的程序。在存活程序中，优先选择 MSE 最小的程序。在 MSE 相同的情况下，保留代码长度最短、FLOPs 最少的程序，以产生奥卡姆剃刀效应。

## 6. 任务要求 (Deliverables)
请用 Python（可使用 `numpy`，不依赖现成复杂 GP 库，手写轻量级框架）实现上述 MWE 的完整核心逻辑：
1.  定义所有 Stacks 的数据结构。
2.  实现带有 FLOPs 拦截机制的 `MIMO_Push_VM` 及其完整的原子指令集执行逻辑。
3.  实现外围的 `StackDecoder`，其中优先级评分被代理给 `VM.run(program)`。
4.  实现一个极简的进化引擎（初始化随机指令序列 -> 交叉/变异 -> 在 MIMO 评估器上测算 Fitness -> 优胜劣汰）。
5.  打印出每一代的最优程序的代码序列（Instruction Sequence）和 Fitness 变化。

**提示词工程师附言：** 请极其严格地遵守“零高阶先验”原则。不要给机器提供任何用于通信计算的“特权函数”。我们期待在输出的控制台中，看到指令序列从无意义的随机噪音，随着代数迭代，慢慢收敛为能够利用上三角阵 $\mathbf{R}$、自动截取子树计算局部距离的优雅结构。
